 ЧАСТЬ 1. ВЫБОР ТЕХНОЛОГИЙ (СТЕК)
================================================================================
1. Язык: Dart (фреймворк Flutter). Почему?
   - Один код под всё.
   - Горячая перезагрузка (hot reload) работает даже в облачных IDE.
   - Есть готовые AR-плагины, которые скрывают сложность ARCore [citation:2][citation:6].

2. AR Плагин для Flutter: augen (версия 0.3.1+)
   - Позволяет работать с ARCore без написания Java/Kotlin.
   - Умеет находить плоскости (стол, пол), ставить якоря (объекты),
     загружать 3D-модели (GLB, GLTF, OBJ) [citation:2].
   - Есть стримы (planesStream) для получения данных о найденных плоскостях.
   - Альтернатива: ar_flutter_plugin (более старый, но с большим комьюнити) [citation:6].

3. 3D Вьювер (Виртуальная область):
   - Сам AR-плагин (augen) не нужен, когда мы просто смотрим на виртуальную сцену.
   - Для просмотра области (без камеры) нужен другой виджет.
   - Вариант 1: model_viewer_plus - Позволяет загружать и крутить 3D-модели (OBJ/GLTF)
        прямо в приложении. Идеально для "режима просмотра области".
   - Вариант 2: flutter_3d_controller - Более мощный, с поддержкой анимаций.

4. Хранение данных:
   - Проекты и их состав: SQLite (встроенный в Flutter через sqflite).
   - 3D-модели (файлы .obj или .glb): Сохраняем в папку приложения на телефоне.
        Путь: getApplicationDocumentsDirectory().

5. Экспорт в OBJ/STL:
   - Это сложная задача. Готовых библиотек под Flutter мало.
   - Стратегия: Сохранять информацию о позициях и размерах объектов в JSON.
        Для экспорта нужно будет генерировать OBJ файл "на лету" (вручную писать
        текст OBJ формата: вершины и грани). Это реализуемо, но требует математики.
   - Упрощение: На первых этапах экспорт делаем в JSON, а потом ищем
        конвертер JSON-to-OBJ для Android, либо просим помощи у ИИ в написании
        функции генерации простых примитивов (кубов, цилиндров) в OBJ.

ЭТАП 0: "Hello, AR!" (1-2 дня)
--------------------------------
- Задача: Настроить проект Flutter, подключить плагин `augen`.
- Реализация: Создать экран, на котором просто запускается камера,
  ищутся плоскости и рисуется сетка. Никакого функционала.
- Тест: Убедиться, что приложение не падает, видит пол/стол.
- Ссылка: Пример кода инициализации есть в документации `augen` [citation:2].

ЭТАП 1: "Линейка" (Точки и линии) (1 неделя)
--------------------------------------------
- Задача: Дать пользователю ставить точки на плоскости и рисовать линии.
- Реализация:
   - При тапе по экрану делаем `hitTest`. Если попали в плоскость, ставим
     в эту точку якорь (ARAnchor) и визуализируем его простой сферой [citation:2][citation:6].
   - Когда поставлено 2, 3, 4 точки, соединяем их линиями.
   - Как нарисовать линию? Самый простой способ - поставить цилиндр,
     повернутый и масштабированный по длине между точками. Это сложно.
     УПРОЩЕНИЕ: Пока просто сохраняем координаты точек и выводим расстояние
     между последними двумя точками на экране (как рулетка).
- Цель: Научиться получать координаты точек касания и работать с ними.

ЭТАП 2: "Проекты и примитивы" (2 недели)
----------------------------------------
- Задача: Создать меню проектов и возможность сохранять простые боксы.
- Реализация:
   1. Меню: Экран со списком проектов (SQLite) и кнопкой "Новый проект".
   2. Новый проект: При создании переходим в AR-режим.
   3. Добавление объекта: Пользователь ставит 4 точки по углам стола.
        Приложение вычисляет центр и размеры (ширина, длина).
   4. Появляется кнопка "Добавить параллелепипед". По нажатию в виртуальную
        сцену (пока что просто в AR-пространство) помещается полупрозрачный куб
        с этими размерами.
   5. Сохранение: Координаты и размеры куба сохраняем в БД проекта.
- Интерфейс: Добавить кнопку "Завершить сканирование", которая возвращает в меню.

ЭТАП 3: "Виртуальная область" (2-3 недели)
------------------------------------------
- Задача: Отделить сканирование от просмотра. Создать 3D-вьювер.
- Реализация:
   1. В проекте появляется кнопка "Открыть виртуальную область".
   2. Это новый экран, где нет камеры, а используется `model_viewer_plus`.
   3. При входе в область, приложение считывает из БД проекта все сохраненные
        объекты (их размеры и позиции) и отображает их как примитивы (кубы,
        цилиндры) в центре сцены.
   4. Добавить управление камерой (вращение, масштабирование) средствами
        `model_viewer_plus`.
- Усложнение: Если мы сохранили стол как куб 1.5x1.0x0.1м, то в виртуальной
        области мы должны увидеть этот куб. Его положение относительно других
        объектов пока определяем просто "на полу" сцены, выстраивая их в ряд.
        Позже добавим возможность двигать их пальцем.

ЭТАП 4: "Сложные формы и цилиндры" (3-4 недели)
-----------------------------------------------
- Задача: Добавить режимы "цилиндр" и "сложный контур".
- Реализация:
   1. В AR-режиме добавляем переключатель: "Бокс", "Цилиндр", "По точкам".
   2. Цилиндр:
        - Пользователь ставит 2 точки (края диаметра) на плоскости (основание).
        - ARCore дает координаты. Вычисляем радиус и центр окружности.
        - Затем пользователь ставит точку на верхнем крае ведра (высота).
        - Создаем цилиндр с найденным радиусом и высотой.
   3. Сложный контур (полигон):
        - Пользователь обходит объект и ставит точки по краям.
        - Приложение соединяет их ломаной линией.
        - Потом "выдавливаем" эту фигуру на заданную высоту (вводится вручную).
        - Это уже будет сложная 3D-модель. Для ее хранения придется генерировать
          меш (сетку из треугольников). Это вызов.
- Экспорт: На этом этапе уже можно пробовать экспортировать эти сложные
  модели в OBJ, так как у нас есть их геометрия.

ЭТАП 5: "Экспорт и импорт" (1-2 недели)
---------------------------------------
- Задача: Реализовать кнопку "Экспортировать проект в OBJ".
- Реализация:
   1. Проходим по всем объектам проекта (кубы, цилиндры, меши).
   2. Для каждого объекта генерируем текст в формате OBJ:
        - 'v ' для вершин.
        - 'f ' для граней.
   3. Объединяем всё в один файл.
   4. Сохраняем файл в папку Downloads или Documents телефона.
   5. Добавить возможность импорта OBJ обратно в проект (загрузка файла
        и размещение его как объекта в сцене). Это открывает безграничные
        возможности: можно скачать модель стула с Shapeways и "поставить"
        его в свою виртуальную комнату.

// 1. ИНИЦИАЛИЗАЦИЯ AR ВО Flutter (с плагином augen)
import 'package:augen/augen.dart';
import 'package:vector_math/vector_math_64.dart' show Vector3;

class ARScreen extends StatefulWidget {
  @override
  State<ARScreen> createState() => _ARScreenState();
}

class _ARScreenState extends State<ARScreen> {
  AugenController? _controller;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: AugenView(
        onViewCreated: _onARViewCreated,
        config: ARSessionConfig(
          planeDetection: true, // ВКЛЮЧАЕМ ПОИСК ПЛОСКОСТЕЙ [citation:2]
          lightEstimation: false,
        ),
      ),
    );
  }

  void _onARViewCreated(AugenController controller) {
    _controller = controller;
    _initializeARSession();
    _listenToPlanes();
  }

  Future<void> _initializeARSession() async {
    await _controller!.initialize(ARSessionConfig(planeDetection: true));
    // Подписываемся на нажатия для hitTest
    _controller!.onPlaneOrPointTap = _handleTap;
  }

  void _listenToPlanes() {
    _controller!.planesStream.listen((List<ARPlane> planes) {
      for (var plane in planes) {
        print('Найдена плоскость: ${plane.type}, центр: ${plane.center}');
        // Можно визуализировать плоскость, но augen часто делает это сам
      }
    });
  }

  // 2. ОБРАБОТКА КАСАНИЯ (HitTest) и ДОБАВЛЕНИЕ МОДЕЛИ
  Future<void> _handleTap(ARTapGesture tap) async {
    // Выполняем hitTest по экрану
    final List<ARHitTestResult> hits = await _controller!.hitTest(tap.x, tap.y);

    if (hits.isNotEmpty) {
      // Берем первое попадание в плоскость
      ARHitTestResult? hit = hits.firstWhere(
        (h) => h.type == ARHitTestResultType.plane,
        orElse: () => hits.first,
      );

      // Создаем якорь (Anchor) в этом месте, чтобы объект не "уплывал" [citation:2]
      final ARAnchor? anchor = await _controller!.addAnchor(hit.position);

      if (anchor != null) {
        // Добавляем 3D модель (простая сфера для теста)
        final ARNode node = ARNode(
          id: 'my_sphere_${DateTime.now().millisecondsSinceEpoch}',
          type: NodeType.sphere, // Можно заменить на NodeType.modelFromAsset
          position: Vector3(0, 0, 0), // Относительно якоря
          scale: Vector3(0.1, 0.1, 0.1),
        );
        await _controller!.addNode(node, anchorId: anchor.id);
        print('Объект добавлен в якоре $anchor');
      }
    }
  }

  // 3. ЗАГРУЗКА ПОЛЬЗОВАТЕЛЬСКОЙ 3D МОДЕЛИ (например, .glb)
  Future<void> _loadCustomModel(String assetPath) async {
    // Файл .glb должен лежать в папке assets и быть прописан в pubspec.yaml
    final ARNode customNode = ARNode.fromModel(
      id: 'custom_object',
      modelPath: assetPath, // 'assets/my_chair.glb'
      position: Vector3(0, 0, -1.5),
      scale: Vector3(0.5, 0.5, 0.5),
    );
    await _controller!.addNode(customNode);
  }
}
// КОНЕЦ ФРАГМЕНТА

 ЧАСТЬ 4. РЕШЕНИЕ ПРОБЛЕМ
================================================================================
1. "Приложение тормозит":
   - Уменьши количество полигонов в 3D-моделях.
   - В конфигурации ARSessionConfig отключи `depthData` и `lightEstimation`,
     если они не нужны [citation:2].
   - Snapdragon 888 мощный, но нагрев будет. Делай паузы в сканировании.

2. "ARCore не видит плоскости":
   - Плохо освещение. Включи свет.
   - Поверхность слишком однородная (белая стена). Нужны визуальные ориентиры.
   - Двигай телефон плавно, чтобы ARCore набрал "особые точки" (feature points) [citation:1].

3. "Как сохранить объекты между сессиями?":
   - В ARCore для Jetpack XR есть Persisted Anchors, но в `augen` их поддержка
     может отсутствовать [citation:7].
   - Наш путь: Сохраняем не якоря, а ПОЗИЦИИ (Vector3) и РАЗМЕРЫ объектов в БД.
     При следующем запуске проекта заново создаем эти объекты на сохраненных позициях.
